# 선행기술 발굴 중심 검색식 생성 프롬프트

첨부된 PDF 특허 문서를 분석하여 선행기술 발굴 및 무효 자료 검색에 특화된 검색식을 생성해주세요.

## 선행기술 분석 요구사항

1. **선행기술 범위 정의**:
   - 본 특허의 우선일 이전 공개된 모든 관련 기술
   - 직접적 선행기술 vs 간접적 참조 기술
   - 특허 문헌 vs 비특허 문헌 (논문, 표준, 제품 등)
   - 동일 발명자/출원인의 이전 출원들

2. **무효 포인트 분석**:
   - 각 청구항별 무효 가능성 평가
   - 신규성(Novelty) 저해 요소
   - 진보성(Non-obviousness) 저해 요소  
   - 결합 가능한 복수 선행기술 조합
   - 공지공용 기술 및 관용 기술

3. **검색 전략** (7개 생성):
   - **Exact Prior Art**: 직접적 선행기술 (우선일 이전)
   - **Inventor's Previous Work**: 동일 발명자의 이전 작업
   - **Assignee's Portfolio**: 동일 출원인의 관련 특허들
   - **Academic Literature**: 학술 논문 및 연구 자료
   - **Standards & Publications**: 표준 문서 및 공개 자료
   - **Combination Approach**: 복수 선행기술 조합
   - **Obvious Variants**: 자명한 변형 기술들

## 선행기술 검색 전략

### 시간적 제약 강조
- `before:우선일`: 우선일 이전 문헌만 검색
- `after:1990 before:우선일`: 특정 기간 내 선행기술
- 출원일/공개일 기준 정밀 필터링

### 청구항별 정밀 분석
- `CL=(청구항1의 핵심요소)`: 독립청구항 중심
- `AB=(종속청구항의 추가요소)`: 세부 기술요소
- 청구항 해석 관점에서의 키워드 선택

### 발명자/출원인 추적
- `inventor:"동일발명자"`: 연속성 있는 연구 추적
- `assignee:"동일회사"`: 기업 내부 기술 축적 확인
- 기술 이전 및 인수합병 이력 반영

## 출력 형식

```json
{
  "patent_info": {
    "publication_number": "특허번호",
    "title": "특허 제목",
    "priority_date": "우선일",
    "filing_date": "출원일", 
    "publication_date": "공개일",
    "assignee": "출원인",
    "inventors": ["발명자1", "발명자2"],
    "main_claims": ["독립청구항1 요약", "독립청구항2 요약"],
    "key_elements": ["핵심기술요소1", "핵심기술요소2"],
    "novelty_points": ["신규성포인트1", "신규성포인트2"],
    "keywords": ["선행기술키워드1", "선행기술키워드2"]
  },
  "search_queries": [
    {
      "strategy": "Exact Prior Art - 우선일 이전 직접적 선행기술 검색",
      "query": "시간 제한 있는 정밀 검색식", 
      "expected_scope": "broad/medium/narrow",
      "target_claims": ["대상 청구항 번호"],
      "invalidity_type": "novelty/obviousness/enablement",
      "time_constraint": "before:우선일",
      "rationale": "이 검색식이 어떤 무효 포인트를 공략하는지 설명"
    }
  ]
}
```

## 특별 지침

1. **시간적 정확성**: 우선일/출원일 기준 엄격한 시간 제한
2. **청구항 중심**: 각 청구항의 기술적 특징에 맞춘 검색
3. **결합 가능성**: 복수 문헌 조합으로 진보성 부정 가능성
4. **범위의 적절성**: 너무 좁거나 넓지 않은 적정 검색 범위
5. **다각적 접근**: 특허/비특허/학술/상용화 등 다양한 소스
6. **법적 관점**: 무효심판/재심사 실무에서 실제 사용 가능한 수준
7. **증거력 고려**: 검색 결과의 법적 증거 능력 및 신뢰성 확보

## 무효 분석 체크포인트

- [ ] 각 독립청구항별 신규성 검토
- [ ] 진보성 부정을 위한 주지관용기술 확인  
- [ ] 동기 결합 가능한 복수 선행기술 조합
- [ ] 발명의 효과 대비 선행기술의 시사점
- [ ] 출원 경위 및 보정 과정에서의 권리화 한계
